# 美化配置文件结构说明
基于规则替换进行文本美化。  
部分代码与设计来自[Faust Launcher](https://github.com/f0lkskill/FaustLauncher)，基于MIT协议使用，让我们感谢他的开拓。  

如果你想要使用美化功能，那么您只需要启用您想要启用的规则集，然后保存设置即可。  

如果您想要自定义美化规则，请阅读下文。

# 自定义美化规则文档

## 1. 配置文件顶层结构

配置文件是一个数组，每个元素代表一个“规则集”（ruleset），包含该规则集的名称、描述和具体规则列表。

```json
[
    {
        "name": "规则集名称",
        "desc": "规则集描述",
        "rules": [ ... ]   // 具体规则数组
    },
    ...
]
```

## 2. 规则（rule）对象结构

每个规则对象定义了一条或多条操作，用于匹配特定的 JSON 路径并应用修改。规则对象包含以下字段：

| 字段名    | 类型             | 必填 | 描述 |
|-----------|------------------|------|------|
| `aimFile` | 字符串           | 否   | 正则表达式，用于筛选文件名。只有文件名匹配此表达式的 JSON 文件才会应用该规则。 |
| `aim`     | 字符串           | 是   | 目标路径模式。有两种用法：<br>① 无 `trigger` 时：为正则表达式，匹配要处理的 JSON 路径。<br>② 有 `trigger` 时：为路径模板，可根据触发路径转换生成新路径。 |
| `trigger` | 对象             | 否   | 触发条件。如果存在，则必须先满足触发条件，才会根据 `aim` 生成新路径并执行操作。 |
| `action`  | 数组             | 是   | 操作列表。每个元素是一个操作对象，支持两种类型：文本替换和颜色渐变。多个操作按顺序依次执行。 |

### 2.1 `aimFile` 字段
- 类型：字符串（正则表达式）
- 作用：限定规则适用的文件。例如 `"Skill.*\\.json$"` 匹配所有以 `Skill` 开头、以 `.json` 结尾的文件。
- 如果不提供，则规则会应用于所有 JSON 文件。

### 2.2 `aim` 字段
#### 无 `trigger` 时
- 类型：正则表达式
- 作用：匹配扁平化后的 JSON 路径。路径以点分隔，例如 `"dataList\\.\\d+\\.levelList\\.\\d+\\.desc"`。
- 所有匹配该路径的值都会应用 `action` 中的操作。

#### 有 `trigger` 时
- 类型：路径模板（点分隔字符串，可包含特殊标记 `[back]`）
- 作用：根据触发匹配到的源路径，转换得到新的目标路径。然后对该目标路径的值应用操作。
- 转换规则：
  - 模板由点分隔的多个部分组成。
  - `[back]` 表示回退一级（移除源路径的最后一个部分）。
  - 其他部分直接追加到路径末尾。
  - 纯数字部分（如 `123`）会被自动转换为整数索引，用于列表访问。
- 示例：
  - 源路径：`dataList.0.levelList.0.desc`
  - `aim`: `"[back].name"`
  - 转换后路径：`dataList.0.levelList.0.name`

### 2.3 `trigger` 对象
`trigger` 用于设置触发条件，包含两个字段：

| 字段名 | 类型   | 描述 |
|--------|--------|------|
| `aim`  | 字符串 | 正则表达式，用于匹配扁平化后的 JSON 路径。 |
| `re`   | 字符串 | 正则表达式，用于匹配 `aim` 路径对应的值（必须是字符串）。 |

只有当某个路径匹配 `aim` 且该路径的值匹配 `re` 时，才会触发该规则。

### 2.4 `action` 数组
`action` 是一个操作列表，每个操作可以是以下两种类型之一：

#### 文本替换操作
- 必须包含 `"from"` 和 `"to"` 两个字段。
- `from`：正则表达式，用于匹配文本中需要被替换的部分。
- `to`：替换后的字符串（可包含反向引用 `\1`、`\2` 等）。
- 示例：`{ "from": "大于", "to": ">" }`

#### 颜色渐变操作
- 必须包含 `"rate"` 字段，可选，默认值为 `2.0`。
- `rate`：控制渐变程度的数值（具体效果由底层函数 `process_dlg_text` 实现）。
- 示例：`{ "rate": 1.0 }`

两种操作可以混合使用，按数组中顺序依次应用。

## 3. 路径表示与匹配

在规则中，JSON 内部路径使用点分隔的字符串表示，例如：
- `dataList.0.levelList.0.desc`
- `dataList.0.coinlist.0.coindescs.0.desc`

数字部分表示数组索引。正则表达式中的点需要转义（`\\.`），例如 `"dataList\\.\\d+\\.levelList\\.\\d+\\.desc"`。

## 4. 处理流程简述

1. 读取所有规则集，合并所有 `rules` 到一个列表。
2. 遍历指定语言包目录下的所有 JSON 文件。
3. 对于每个文件，筛选出 `aimFile` 匹配的规则。
4. 将 JSON 数据扁平化为键值对，键为路径元组，值为原始值。
5. 对每条规则：
   - 如果有 `trigger`，则先匹配触发路径和值，对每个触发成功的源路径，根据 `aim` 模板生成目标路径，然后对目标路径的值应用 `action`。
   - 如果没有 `trigger`，则直接用 `aim` 正则匹配路径，对所有匹配的路径值应用 `action`。
6. 将更新后的值写回原始 JSON 结构，并保存文件。

## 5. 示例解析

以下取自 `build-in-fancy.json` 的部分规则示例：

### 示例 1：技能文本美化
```json
{
    "name": "技能文本美化(FL Like)",
    "rules": [
        {
            "aimFile": "Skill.*\\.json$",
            "aim": "dataList\\.\\d+\\.levelList\\.\\d+\\.desc",
            "action": [
                { "from": "大于", "to": ">" },
                { "from": "自身", "to": "<u><color=#7C5738>自身</color></u>" },
                { "from": "体力", "to": "<u><color=#61DA61>体力</color></u>" }
            ]
        }
    ]
}
```
- 适用于文件名匹配 `Skill*.json` 的文件。
- 路径 `dataList.<数字>.levelList.<数字>.desc` 下的文本，将依次进行文本替换：`大于` → `>`，`自身` → 带下划线和颜色的 HTML 标签，`体力` → 带下划线和绿色标签。

### 示例 2：EGO文本渐变
```json
{
    "name": "EGO文本渐变(FL Like)",
    "rules": [
        {
            "aimFile": "Skills_Ego_Personality-.*\\.json$",
            "trigger": {
                "aim": "dataList\\.\\d+\\.levelList\\.\\d+\\.desc",
                "re": "指定"
            },
            "aim": "[back].name",
            "action": [
                { "from": "^(.*)$", "to": "<color=#ff0000>\\1</color>" },
                { "rate": 0.5 },
                { "from": "^(.*)$", "to": "<b><i>\\1</i></b>" }
            ]
        }
    ]
}
```
- 适用于 `Skills_Ego_Personality-*.json` 文件。
- 触发条件：路径 `dataList.<数字>.levelList.<数字>.desc` 的值包含“指定”。
- 触发后，将源路径回退一级（去掉 `.desc`），再拼接 `.name`，得到目标路径（例如 `dataList.0.levelList.0.name`）。
- 对目标路径的值依次应用操作：
  1. 用红色颜色标签包裹整个文本。
  2. 应用渐变效果（rate=0.5）。
  3. 用粗体斜体标签包裹整个文本。
