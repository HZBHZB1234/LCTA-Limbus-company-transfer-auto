# 翻译工具

将游戏文本（韩文、日文、英文）自动翻译成中文。支持多种翻译服务，包括在线翻译 API 和本地 LLM。

## 使用流程

1. **配置翻译服务**：先在 **配置汉化API** 页面配置好翻译服务的 API 密钥（如 Deepseek、百度翻译等）。
2. **设置翻译选项**：根据需求启用各项翻译选项（详见下方说明）。
3. **选择翻译服务**：在下拉框中选择已配置的翻译服务。
4. **开始翻译**：点击底部"开始翻译"按钮，在弹出的进度窗口中等待完成。

## 翻译选项

### 专有名词相关

- **启用专有名词 (LLM翻译专用)**：翻译时参考专有名词表，提高人名、地名等术语的翻译一致性。在 LCTA-AU 中默认开启。如果出现大量匹配错误请关闭。
- **自动抓取专有词汇**：翻译过程中自动重新提取专有词汇。如需使用自定义专有名词文件，请关闭此选项并在下方手动指定文件路径。
- **专有名词 JSON 路径**：关闭自动抓取后，可手动指定自定义的专有名词 JSON 文件。

### LLM 翻译专用选项

- **启用人物角色标识**：为 LLM 翻译添加上下文标记，帮助模型理解当前文本属于哪个角色，提升角色对话翻译准确性。
- **启用状态效果标识**：为 LLM 翻译添加状态效果标记，帮助模型理解技能描述中的游戏机制术语。

### 高级选项

开启"启用高级选项"后，可手动指定各语言的源文本路径：

- **韩文文本路径**：原始韩文游戏文本所在的文件夹。
- **日文文本路径**：日文游戏文本所在的文件夹。
- **英文文本路径**：英文游戏文本所在的文件夹。
- **中文文本路径**：参考中文文本所在的文件夹（通常为零协会汉化文本）。如果未安装零协会汉化，需要手动指定此路径。
- **文件名存在前缀**：按带前缀的文件名格式处理源文件。

> 通常情况下无需手动设置以上路径，程序会自动检测。仅在自动检测失败或需要特殊处理时才需手动配置。

### 其他选项

- **转储过程内容以供分析**：将请求文本与响应文本输出至日志，方便调试和问题排查。

## 翻译服务配置

- **翻译服务**：在下拉列表中选择已配置好的翻译服务（如 DeepL、Google、本地 LLM 等）。
- **失败时进行相反重试 (LLM翻译专用)**：LLM 返回格式错误时切换请求格式重试。**建议开启**，否则可能出现较多漏翻。
- **使用文本格式进行请求 (LLM翻译专用)**：使用自然语言请求并以文本格式而非 JSON 格式返回。**建议关闭**，否则可能出现较多错位。
- **源语言 (除LLM翻译外使用)**：选择要翻译的源语言 —— 英文、日文或韩文。LLM 翻译无需选择此项。

## 开始翻译

配置完成后点击 **开始翻译** 按钮启动翻译任务。翻译过程中：

- 进度窗口会显示当前翻译状态和进度百分比。
- 支持**暂停 / 继续**翻译任务。
- 支持**取消**翻译任务（已翻译的部分会保留）。
- 可**最小化**进度窗口，在右下角查看迷你进度条。
